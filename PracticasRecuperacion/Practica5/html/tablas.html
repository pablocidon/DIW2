Tablas convencionales

Las tablas en HTML son estructuras que están formadas por varias filas, cada una de las cuales tiene un conjunto de celdas.

Las tablas se crean con las etiquetas <table> </table>. Dentro de las etiquetas de apertura y cierre, inmediatamente después se puede colocar una etiqueta <caption> </caption> que incluya el título de la tabla. A continuación, se identifica cada fila de la tabla con las etiquetas <tr> </tr>. Por último, cada celda dentro de una fila se identifica con las etiquetas <td> </td>. A continuación puedes ver un ejemplo: 

    <table border="1">
        <caption>Título de la tabla</caption>
        <tr>
            <td>Fila1-celda1</td>
            <td>Fila1-celda2</td>
            <td>Fila1-celda3</td>
            <td>Fila1-celda4</td>
        </tr>
        <tr>
            <td>Fila2-celda1</td>
            <td>Fila2-celda2</td>
            <td>Fila2-celda3</td>
            <td>Fila2-celda4</td>
        </tr>
        <tr>
            <td>Fila3-celda1</td>
            <td>Fila3-celda2</td>
            <td>Fila3-celda3</td>
            <td>Fila3-celda4</td>
        </tr>
    </table>

El resultado sería:

Título de la tabla
Fila1-celda1 Fila1-celda2 Fila1-celda3 Fila1-celda4
Fila2-celda1 Fila2-celda2 Fila2-celda3 Fila2-celda4
Fila3-celda1 Fila3-celda2 Fila3-celda3 Fila3-celda4

Atributos:

  • width, anchura total de la tabla en píxeles o porcentaje.
  • border, que indica la anchura del borde de la tabla en píxeles (por defecto vale 0, por lo que no se muestra el borde) y los atributos.
  • rules, que indican qué líneas de división entre celdas serán visibles.
    • none: ninguna, es el valor por defecto.
    • groups: solo entre grupos de filas o grupos de columnas.
    • rows, cols: solo entre filas, solo entre columnas
    • all: todas.
  • frames, que indica qué lados de la tabla serán visibles.
    • void: ninguna, es el valor por defecto.
    • above, below, lhs, rhs: solo arriba, solo abajo, solo izquierda, solo derecha
    • box: todos los lados.
  • cellspacing, especifica el espacio entre celdas.
  • cellpadding, especifica el espacio entre el bore de una celda y su contenido.

Contenido de <table>:

  • <tr>, para crear una fila dentro de una tabla. La etiqueta final es opcional.
    • align, indica la alineación del texto a nivel de fila. align=”left|right|center|middle|bottom”.
    • valign, indica la alineación del texto verticalmente a nivel de fila. valign=”top|middle|bottom|baseline”.
    • bgcolor, En desusuo
  • <td>,<th>, para crear una celda dentro de una fila. ( es celda de cabecera).
    • align, indica la alineación del texto a nivel de celda. align=”left|right|center|justify”.
    • valign, indica la alineación del texto verticalmente a nivel de celda. valign=”top|middle|bottom”.
    • rowspan, indica el número de filas que ocupa la celda (combinar filas).
    • colspan, indica el número de columnas que ocupa la celda (combinar columnas).
    • bgcolor, En desusuo

Prueba el siguiente código y guárdalo como Ejemplo_10.html

    <table border="3" cellspading="0" cellpadding="0" width="230">
        <tr>
            <td colspan="3" height="10"></td>
        </tr>
        <tr>
            <td width="10" height="44"></td>
            <td width="210" align="center">
                <strong>HyperText Markup Language</strong>
            </td>
            <td width="10" height="44"></td>
        </tr>
        <tr>
            <td colspan="3" height="10"></td>
        </tr>
    </table>

El resultado sería:

HyperText Markup Language

Celdas de cabecera

Es posible sustituir algunos elementos <td> por elementos <th>. Estos elementos hacen referencia a celdas de cabecera, y como tales aparecerán destacadas del resto (suelen aparecer en negrita y centradas).

Prueba el siguiente código y guárdalo como Ejemplo_11.html

    <table border="1" border="3" cellspading="0" cellpadding="5" width="300">
        <tr>
            <th>Cod</th>
            <th>Nombre</th>
            <th>Significado</th>
        </tr>
        <tr>
            <td>1</td>
            <td>SGML2</td>
            <td>Standard Generalizaed Markup Language</td>
        </tr>
        <tr>
            <td>2</td>
            <td>HTML</td>
            <td>HyperText Markup Language</td>
        </tr>
        <tr>
            <td>3</td>
            <td>XML</td>
            <td>Extensible Markup Language</td>
        </tr>
    </table>

Cod Nombre Significado
1 SGML2 Standard Generalizaed Markup Language
2 HTML HyperText Markup Language
3 XML Extensible Markup Language

Grupos de columnas

Los grupos de columnas permiten hacer referencia a un grupo de columnas dentro de una tabla. Para ello se utilizan las etiquetas <colgroup> y <col>. Estas etiquetas son interesantes porque permiten definir la anchura de un conjunto de columnas a la vez, evitando tener que definir la anchura de cada columna por separado. Se deben incluir al principio del código de la tabla, justo después de la etiqueta <caption> (si existiera).

Ambas etiquetas se suelen utilizar con los atributos width (que indica la anchura de las columnas en píxeles o porcentaje), align (que indica la alineación de las columnas) y span (que indica el número de columnas que se abarca en el conjunto indicado).

¡Ojo! Estas etiquetas no funcionan correctamente en Firefox (la anchura de las columnas sí que parece funcionar, mientras que la alineación no).

La etiqueta <colgroup> se utiliza para indicar un conjunto de columnas. Así, si queremos indicar que las 5 primeras columnas de una tabla deben tener anchura 20 píxeles y las 3 siguientes 10 píxeles, incluiremos el siguiente código :

    <colgroup span="5" width="20">
    </colgroup>
    <colgroup span="3" width="10">
    </colgroup>

La etiqueta <col> se suele utilizar para especificar un determinado número de celdas dentro de un elemento <colgroup>. Normalmente se utiliza para que un determinado conjunto de celdas reciba un atributo especial además de los especificados en <colgroup>. Así, si queremos indicar que las 5 primeras columnas de una tabla deben tener anchura 20 píxeles pero la columna 3, además de tener el ancho de 20 píxeles, debe recibir algún tratamiento posterior (normalmente mediante hojas de estilo), indicaremos lo siguiente: 

    <colgroup span="5" width="20">
        <col span="2">
        <col align="center">
        <col span="2">
    </colgroup>

En el ejemplo anterior, las 5 primeras columnas tendrán 20 píxeles de ancho. Dentro de <colgroup> se detallan las demás columnas: las dos primeras quedan igual; la número 3 alinea su contenido al centro; y las dos últimas, quedan también igual. ¡Ojo! el número de columnas que se indique en la etiqueta <colgroup> deberá ser igual a la suma de los números de columnas indicados por las etiquetas <col> de su interior.

Grupos de filas: cabecera, cuerpo y pie de tablas

En ocasiones, es interesante dividir las filas de la tabla en diferentes partes organizativas. HTML contempla tres: cabecera, cuerpo y pie. Esta división se utiliza para diferenciar dichos campos y tratarlos por separado. Así, por ejemplo, en tablas muy largas que deban ser imprimidas en varias hojas, el navegador imprimirá la cabecera y el pie de la tabla en cada una de las hojas.

Las etiquetas que se utilizan para esto son:

  • <thead> </thead> para la cabecera. Debe aparecer en primer lugar. Normalmente, las celdas contenidas en las filas de la cabecera serán del tipo <th> para que queden destacadas.
  • <tfoot> </tfoot> para el pie. Debe aparecer en segundo lugar.
  • <tbody> </tbody> para el cuerpo. Puede haber más de un cuerpo en la misma tabla.

Cada una de dichas etiquetas agrupa un conjunto de filas. A continuación tienes un ejemplo. En él se ha definido un encabezado, un pie y dos cuerpos de la tabla. Para identificar cada uno, se ha utilizado el atributo rules, que indica qué bordes y separaciones aparecerán en la tabla (el valor groups indica que deben separarse los grupos de filas y columnas). 

    <table border="1" rules="groups">
        <caption>Título de la table</caption>
        <thead>
            <tr>
                <th>Fila1_cabecera-celda1</th>
                <th>Fila1_cabecera-celda2</th>
                <th>Fila1_cabecera-celda3</th>
                <th>Fila1_cabecera-celda4</th>
            </tr>
        </thead>
        <tbody>
            <tr>
                <td>Fila1_cuerpo1-celda1</td>
                <td>Fila1_cuerpo1-celda2</td>
                <td>Fila1_cuerpo1-celda3</td>
                <td>Fila1_cuerpo1-celda4</td>

            <tr>
                <td>Fila2_cuerpo1-celda1</td>
                <td>Fila2_cuerpo1-celda2</td>
                <td>Fila2_cuerpo1-celda3</td>
                <td>Fila2_cuerpo1-celda4</td>
            </tr>
        </tbody>
        <tbody>
            <tr>
                <td>Fila1_cuerpo2-celda1</td>
                <td>Fila1_cuerpo2-celda2</td>
                <td>Fila1_cuerpo2-celda3</td>
                <td>Fila1_cuerpo2-celda4</td>

Celdas que abarcan varias filas o columnas

Para hacer que una celda ocupe varias celdas, se utilizan los atributos rowspan y colspan dentro de las etiquetas <td> (o <th>, si se utilizan las celdas destacadas). Con el primero, se consigue que una celda ocupe varias filas; con el segundo, se consigue que una celda ocupe varias columnas.

Las celdas ocupadas por una celda ampliada no se incluirán en el sitio que les toque.
